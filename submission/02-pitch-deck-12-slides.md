## 第 2 页：问题

### 标题

世界杯流量全球化，但运营成本高、时效要求极强

### 要点

- 每场比赛都需要快速整理赛程、写内容、做互动、带转化
- 小团队和独立开发者难以在比赛窗口内完成全部动作
- 许多 AI 工具只能生成文案，无法完成真正的比赛日工作流

### 讲述句

我们解决的是海外足球流量运营效率问题，而不是再做一个体育资讯网站。

## 第 3 页：目标用户与市场

### 标题

服务真实海外足球用户

### 要点

- 海外球迷社区运营者
- 体育创作者
- 品牌赞助运营者
- 独立开发者和小型出海团队

### 重点市场

- 东南亚 / 印尼
- 西语市场
- 全球英语市场

## 第 4 页：解决方案

### 标题

一套从比赛输入到球迷动作的自主工作流

### 流程

1. 选择真实比赛和目标市场
2. GMI 生成多语言比赛日内容
3. 自动挂载投票和互动模块
4. 继续导向球迷房、观赛活动和赞助转化
5. 后台记录全部数据

### 讲述句

这不是简单问答，而是一条完整 AI 自动化工作流。

## 第 7 页：技术架构

### 标题

前端 + 后端 + 数据 + 推理层完整打通

### 结构

- 前端：双语产品页
- 后端：Node.js 自建 HTTP Server
- 数据库：SQLite
- 推理：GMI Cloud Inference Engine
- 赛事数据：TheSportsDB
- 新闻数据：BBC Sport RSS

### 建议配图

- 浏览器 → Server → GMI / Fixtures / SQLite 的结构图
