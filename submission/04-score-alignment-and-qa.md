# MatchBuzz 评分对齐与答辩提纲

## 1. 评分维度对齐

### 技术实现 30 分

评委关注：

- Agent 编排架构是否合理
- GMI Cloud API 使用是否深入
- 系统是否稳定、真实可运行

MatchBuzz 对应点：

- 自建垂直工作流 Agent，而不是单一聊天界面
- GMI 用于首页洞察、页面情报、多语言比赛日内容生成
- 有真实后端、SQLite 持久化、后台监控、可本地备份演示

### 产品完整度 25 分

评委关注：

- 功能是否完整
- 用户体验是否顺畅
- 是否形成完整闭环

MatchBuzz 对应点：

- 首页、赛事页、比赛详情、球迷房、观赛活动、赞助合作、增长页、后台页齐全
- 有真实会员系统、积分系统、投票互动、分享卡片
- 从内容生成到互动再到商业动作形成完整闭环

### 场景契合度 20 分

评委关注：

- 对出海或独立开发者场景是否理解到位

MatchBuzz 对应点：

- 聚焦海外足球内容运营效率问题
- 面向东南亚、西语市场和全球英语用户
- 适用于独立开发者、小团队、球迷社区和创作者

### 创新性 15 分

评委关注：

- 技术方案或商业模式是否有新意

MatchBuzz 对应点：

- 把 AI 生成、足球工具层、社群层、增长层和变现层整合成一体
- 不是单点内容生成，而是比赛日操作系统
- GMI 使用具备可观测性，适合答辩验证

### 商业潜力 10 分

评委关注：

- 市场规模
- 商业模式
- 后续扩展空间

MatchBuzz 对应点：

- 赞助库存
- 观赛活动与票务转化
- 球迷电商与积分体系
- 后续可工具化为 B2B 出海运营产品

## 2. 评委可能追问的问题

### 问题 1：为什么这不是一个普通内容生成器？

建议回答：

因为 MatchBuzz 不是停留在生成文案。它会读取真实赛事，调用 GMI 生成多语言内容，再自动挂载互动模块，并继续把用户导向球迷房、观赛活动和赞助线索页面，后台还会记录完整运营数据，所以它是完整工作流，而不是单次输出。

### 问题 2：出海场景体现在哪里？

建议回答：

产品默认英文界面，支持多语言内容生成，目标市场明确指向东南亚、西语市场和全球英语市场，产品逻辑、流量路径和商业设计都是围绕海外足球用户构建的。

### 问题 3：你们是如何使用 GMI Cloud 的？

建议回答：

GMI 用于首页比赛日洞察、页面情报生成和比赛日内容生成，不是只用于一个文本框。后台还能展示 token 用量和调用 scope，现场可以直接验证。

### 问题 4：为什么不用 LangChain 或 CrewAI？

建议回答：

本次比赛现场稳定性比框架复杂度更重要。我们选择自建轻量工作流架构，减少故障面，同时更容易解释每个模块的职责和输入输出关系。

### 问题 5：商业模式为什么成立？

建议回答：

因为足球流量本身就有极强的赞助和线下观赛转化价值。我们把赞助库存、观赛活动、票务积分抵扣和球迷电商放进了一条闭环里，不依赖单一 SaaS 收费。

### 问题 6：世界杯之后怎么扩展？

建议回答：

世界杯只是最强验证场景。后续可以扩展到欧洲杯、美洲杯、联赛足球，甚至其他体育赛道的流量运营产品。

## 3. 现场答辩策略

如果评委偏技术：

- 强调自建工作流 Agent
- 强调 GMI 深度接入
- 强调后台 token 和调用日志

如果评委偏产品：

- 强调真实可点击闭环
- 强调球迷房、观赛活动、会员体系、赞助页

如果评委偏场景：

- 强调真实海外足球内容运营的高时效、高重复劳动问题

如果评委偏商业：

- 强调赞助库存、票务与观赛、会员积分、电商和 B2B 扩展

## 4. 一句话总结答辩口径

MatchBuzz 不是一个“生成几段足球文案”的工具，而是一套围绕世界杯流量的全球足球内容互动与商业化工作流系统。
