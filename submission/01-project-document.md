# MatchBuzz 项目文档

## 1. 项目简介

### 项目名称

MatchBuzz

### 一句话介绍

一个面向海外足球球迷社区、体育创作者与品牌运营团队的 AI 世界杯内容互动工作流平台。

### 与赛题的关系

MatchBuzz 对应的是 `AI × 出海` 场景。我们解决的不是“再做一个体育资讯页”，而是海外足球流量运营中的真实效率问题：每逢重大赛事，运营者需要在极短时间内完成赛事信息整理、多语言内容生成、互动玩法搭建、社区承接和商业转化设计，这对独立开发者、小团队和出海内容运营者来说成本很高、时效要求很强。

MatchBuzz 把这条链路做成了一套可自主执行的完整工作流系统。

## 2. 问题定义

### 目标用户

- 海外足球球迷社区运营者
- 海外体育短视频 / 社媒创作者
- 希望围绕足球流量做营销的品牌方和赞助方
- 独立开发者或小团队型出海产品运营者

### 用户痛点

1. 一场比赛往往意味着多步重复劳动：
   - 查找真实赛程和背景信息
   - 撰写不同市场、不同语言的赛前 / 赛中 / 赛后内容
   - 设计投票、互动和讨论入口
   - 承接社群和线下观赛活动
   - 连接赞助位、票务和周边转化
2. 足球流量高度时效化，错过比赛窗口，流量就会快速流失。
3. 市面上很多 AI 工具只停留在“生成一段文案”，并没有完成真正的工作流闭环。
4. 海外产品需要英文主界面和全球化逻辑，同时还需要多语言内容能力。

## 3. 解决方案

MatchBuzz 不是问答型产品，而是足球比赛日工作流产品。

它可以自主执行完整链路：

1. 获取真实比赛上下文
2. 识别目标市场与目标语言
3. 基于 GMI Cloud 生成本地化比赛日内容
4. 自动挂载互动模块，例如投票、支持率、讨论点
5. 把用户继续导向球迷房、观赛活动、赞助合作和积分兑换
6. 在后台记录用户行为、运营数据与 GMI 调用数据

## 4. 目标市场

### 核心目标市场

- 东南亚，尤其是印尼
  - 移动端足球用户活跃
  - 短内容与轻互动消费习惯明显
- 西语市场
  - 足球内容需求强
  - 对宿敌叙事、球星话题、赛后复盘接受度高
- 全球英语市场
  - 创作者、品牌团队、国际球迷社区更适合用英语作为操作层

### 为什么这是一个真实出海产品

- 默认界面为英文
- 提供完整中文界面用于双语展示
- 支持多语言输出
- 流量逻辑、产品结构和商业模式都围绕海外足球用户，而不是国内单一用户群

## 5. 产品工作流

### 端到端流程

1. 用户进入英文首页，选择比赛、支持球队、目标市场、输出语言、内容场景和分发渠道。
2. 系统拉取真实足球赛程与足球新闻信息。
3. GMI Cloud 生成：
   - 标题
   - 社媒文案
   - 短视频口播稿
   - 球迷口号
   - 赛后复盘角度
   - 标签
   - 投票问题与选项
4. 系统自动生成比赛日工作区：
   - 实时投票
   - 支持率模块
   - 分享卡片
   - 活动运营队列
5. 用户可继续进入：
   - 球迷房
   - 观赛活动预约
   - 赞助合作页
   - 会员注册与积分体系
6. 后端记录：
   - 活动工作流
   - 会员数据
   - 积分流水
   - 预约记录
   - 赞助线索
   - GMI 调用记录与 token 用量

## 6. 产品功能亮点

### 不是 PPT，而是真实可点击网站

- 英文首页 + 中文切换
- 真实赛事列表页

## 7. 技术架构

### 前端

- 基于 `public/` 目录下的静态产品页面
- 中英文双语页面体系
- 浏览器侧通过接口拉取实时模块
- 以足球产品逻辑组织首页、赛事页、社区页和增长页

### 后端

- 使用 Node.js 自建 HTTP Server
- 主文件为 `server.js`
- 不依赖重型框架，保证 demo 稳定和可控
- 提供内容生成、赛事数据、会员系统、互动投票和变现链路的 API

### 数据层

- SQLite 持久化
- 存储实体包括：
  - members
  - member_sessions
  - member_points_ledger
  - campaigns
  - polls
  - watch_party_reservations
  - watch_party_point_redemptions
  - sponsor_leads
  - llm_call_logs

### 外部服务

- GMI Cloud Inference Engine
  - 用于内容生成和页面情报生成
- TheSportsDB
  - 用于真实足球赛事和比赛详情
- BBC Sport RSS
  - 用于首页真实足球新闻头条

## 8. Agent / 工作流设计

MatchBuzz 并不是一个“输入一句话，让模型回答”的通用聊天机器人，而是一个自建的垂直工作流 Agent 系统。

### Agent 结构

#### 1. 比赛上下文 Agent

职责：

- 获取真实赛事
- 获取足球新闻
- 标准化比赛信息
- 构造比赛快照

核心函数：

- `fetchRealFixturesFeed`
- `fetchFootballNews`
- `buildRemotePromptText`

#### 2. 本地化内容生成 Agent

职责：

- 调用 GMI Cloud
- 生成多语言比赛日内容
- 把输出整理为产品字段

核心函数：

- `createGeneratedContent`
- `generateWithRemoteLlm`
- `callRemoteJsonField`

#### 3. 页面情报 Agent

职责：

- 为首页、赛事页、比赛页、社区页、观赛页、赞助页、增长页生成短标题和页面引导
- 避免页面呈现方式像 PPT，而更像真实足球产品

核心函数：

- `createPageIntel`
- `createDailyBriefing`

#### 4. 社区与商业化 Agent

职责：

- 生成活动工作区
- 驱动球迷房加入流程
- 处理观赛活动预约
- 接收赞助合作线索
- 将互动流量继续导向商业动作

核心函数：

- `createCampaign`
- `joinCommunityRoom`
- `reserveWatchPartyWithPoints`
- `createSponsorLead`

#### 5. 运营与可观测 Agent

职责：

- 记录 GMI 调用
- 统计 token
- 为后台提供实时可验证的模型使用数据

核心函数：

- `recordLlmCallLog`
- `getLlmUsageSummary`
- `getSystemStatus`

### 为什么这满足“完整工作流”

因为系统不是只给建议，而是：

- 获取真实输入数据
- 调用模型完成结构化生成
- 将结果变成产品模块
- 持久化写入后台
- 引导用户进入后续互动和变现环节

这符合比赛要求中“系统应能够自主执行完整工作流程，而非仅提供简单问答或建议”。

## 9. GMI Cloud API 使用说明

### 使用深度

GMI Cloud 被用于多个流程层，而不是单一输出：

- 首页比赛日洞察
- 各页面 page-intel 短情报
- 多语言比赛日内容生成

### 可验证方式

在后台管理页和 `/api/system/status` 中可以看到：

- 调用总次数
- prompt tokens
- completion tokens
- total tokens
- 最近调用记录
- 调用 scope，例如：
  - `generate`
  - `homepage-briefing`
  - `page-intel:growth`

### 说明

这表明 GMI 在产品中承担的是“工作流核心推理层”，不是装饰性调用。

## 11. 使用的开源框架 / 版本说明

### 当前技术路线

本项目当前不依赖 LangChain、CrewAI、AutoGen 等重型 Agent 编排框架，而是采用轻量、自建、垂直领域工作流架构。

### 当前主要栈

- Node.js `>=20`
- Node 内置 `node:sqlite`
- 浏览器原生 `fetch`

### 第三方服务

- GMI Cloud Inference Engine
- TheSportsDB
- BBC Sport RSS

### 为什么不用重型编排框架

因为本次比赛更重要的是：

- 系统可运行性
- 现场稳定性
- 结构清晰可讲解
- 本地备份可用

自建轻量工作流结构更适合比赛现场。
